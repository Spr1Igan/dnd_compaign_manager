# D&D Campaign Manager

**Текущая версия:** `0.13.0`

D&D Campaign Manager — веб-приложение на Laravel для ведения персонажей Dungeons & Dragons 5e. Сейчас основной фокус проекта — удобный лист персонажа: создание, просмотр, редактирование, автоматические расчёты и быстрые изменения игровых значений во время сессии.

## Что уже реализовано

- Регистрация, вход и выход пользователя.
- Главная страница приложения и навигация.
- Список персонажей текущего пользователя.
- Создание, просмотр, редактирование и удаление персонажа.
- Отдельные маршруты персонажей в `routes/characters.php`.
- Стилизованная форма персонажа в духе бумажного character sheet.
- Русскоязычный интерфейс листа персонажа.
- Справочники рас, классов, предысторий, навыков и языков через сидеры.
- Справочники подрас и подклассов персонажа через сидеры.
- Автоподтягивание данных от выбранной расы, класса и предыстории.
- Автоподтягивание бонусов, языков, скорости и особенностей от выбранной подрасы.
- Автоподтягивание особенностей подкласса в зависимости от текущего уровня персонажа.
- Расчёт итоговых характеристик с учётом расовых бонусов.
- Расчёт модификаторов характеристик.
- Расчёт стартовых HP от кости хитов класса и Телосложения.
- Расчёт КД без доспеха как `10 + модификатор Ловкости`.
- Поддержка ручного КД для доспехов, щитов и особых правил.
- Расчёт бонуса мастерства по уровню.
- Расчёт спасбросков с учётом владения класса.
- Интерактивное изменение текущих HP на странице персонажа.
- Интерактивное прибавление и вычитание опыта.
- Подсказки по характеристикам, навыкам, спасброскам, КД и бонусу мастерства.
- Кликабельные подсказки для особенностей персонажа на форме и в листе просмотра.
- Клиентская и серверная валидация числовых полей листа персонажа.
- Кастомное подтверждение удаления персонажа.
- Тесты для ключевой логики КД.
- Переключатель языка интерфейса `RU / EN` в шапке сайта.
- Локализация основных страниц, форм, листа персонажа, игровых терминов и подсказок на русский и английский языки.
- Кнопка «Запомнить меня» на странице входа сохраняет долгую авторизацию.
- На лист персонажа добавлены владения доспехами, оружием и инструментами от класса и предыстории.
- Для персонажа можно вручную добавить дополнительные владения доспехами, оружием и инструментами.

## Стек

- PHP / Laravel
- Blade
- Vite
- CSS
- SQLite для тестов

```bash

```

```bash

```


## Версионирование

Проект использует формат `0.x.y`.

- `0` — проект ещё находится в активной разработке до первой стабильной версии `1.0.0`.
- `x` — добавление новой функции или заметного пользовательского сценария.
- `y` — мелкие исправления, визуальные правки, уточнение расчётов, документация без изменения основной логики.

Примеры:

- `0.4.0` — добавлена новая функция.
- `0.4.1` — исправлена ошибка или улучшена мелкая деталь в рамках версии `0.4`.
- `0.5.0` — добавлен следующий крупный блок функциональности.

## История версий

### `0.13.0`

- Для драконорождённых добавлены отдельные варианты драконьего происхождения как подрасы: белый, бронзовый, зелёный, золотой, красный, латунный, медный, серебряный, синий и чёрный дракон.
- При выборе происхождения подтягиваются конкретный вид урона, форма оружия дыхания и соответствующее сопротивление.
- Старый общий пункт `Особенности драконорождённых` удаляется сидером, чтобы выбор был точным по виду дракона.

### `0.12.1`

- Для драконорождённого добавлен выбираемый вариант подрасы `Особенности драконорождённых`.
- В этот вариант вынесены особенности из книги игрока: драконье происхождение, оружие дыхания и сопротивление урону.
- Базовые особенности у расы сохранены, чтобы уже созданные драконорождённые не потеряли данные.

### `0.12.0`

- Особенности персонажа теперь отображаются как кликабельные плашки.
- Добавлены подсказки к особенностям, которые подтягиваются от расы, подрасы, класса, подкласса и предыстории.
- Динамический блок формы показывает особенности с теми же подсказками, что и готовый лист персонажа.

### `0.11.0`

- Бонусы характеристик теперь учитывают не только расу, но и подрасу.
- Скорость, языки и особенности подрасы автоматически попадают в лист персонажа.
- Особенности подкласса автоматически добавляются в лист только при достаточном уровне персонажа.
- Добавлены миграции, модели, сидеры, локализованные названия и тесты для новой логики.
